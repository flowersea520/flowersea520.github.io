git一些概念

包含了 .git文件的目录就是本地仓库目录，也叫工作区，

在有.git目录文件夹的地方右键git工具（右键后有分支显示，代表进去了git本地仓库），用来操作本地仓库

git工作区中文件的状态

使用git status 命令，仓库git工作区的文件状态（也能看到文件在哪个分支，其中On branch main语句，表示当前在main分支上面）

如图：下面的test文件是刚刚新建的，所以没有到我们的暂存区（未被纳入版本控制），可以用git add *（这里的✳也可以用文件名代替），再次将工作区的目录文件全部纳入版本控制（状态就会显示staged，代表已在暂存区了）

使用git add * 后，即可显示staged（代表已在暂存区）

Changes to be committed语句代表：需要提交commit的文件（因为已经在暂存区了，所以用commit命令即可提交到本地仓库）

git常用命令：

git reset 命令，将暂存区的文件取消暂存或者是切换到指定版本

使用git reset + 文件名.后缀名，取消暂存区（该指定文件显示为跟踪状态）

git commit提交到本地仓库命令，这个-m 是添加一个注释信息，所以后面要跟一个字符串（这个注释信息会在日志当中）

使用git commit -m “注释” * (也可以加特定文件名) 命令将文件全部提交

其中：On branch main

(use “git push” to publish your local commits)：

代表：在分支干管上
您的分支比“origin/main”提前1次提交。
（使用“git push”发布本地提交）

使用git log 命令可以查看提交（必须是在commit后，提交到本地仓库后git log才会显示版本信息）的版本

使用git reflog show命令,可以查看完整的提交历史，所有的commit_id都会显示，只要有commit_id，我们就能恢复任意版本的代码，在各版本之间来回穿梭。

git 远程仓库操作命令

git remote：查看远程仓库：origin （默认远程仓库简称）– 由于是克隆下来的，所以才能看到远程仓库名

git remote -v查看远程仓库对应的网路地址：（每一个远程仓库都有对应的网络地址）

如果不是远程仓库克隆下来的，则使用git remote不会显示对应的远程仓库

git remote add <简称> URL将本地仓库添加到远程仓库：一般这里的简称用origin

将本地麦吉项目初始化（git init）为本地仓库后，添加远程仓库命令：添加成功后使用git remote查看远程仓库

git push 【remote-name–就是origin】【branch-name– 分支名，master或者其他分支】将本地仓库内容推送到远程仓库

其中remote-name可以使用：git remote命令去查看，一般都是origin

代码推送到远程仓库演示：

先查看本地仓库文件状态，是否有未添加到暂存区的文件git add或者为提交到(git commit)本地仓库的文件

提交到本地仓库，

git status显示本地仓库所有的文件都已经提交

git push origin master: 推送到仓库的主分支上面去

如果创建了一个文件，或者新更新（编辑）了一个文件，就要重新添加到暂存区，然后提交commit到本地仓库

如果线上代码（远程仓库）和线下（本地仓库）代码不一致就会push推送报错（之所以出现这个原因，是因为我在线上生成、编辑了README.md文件，而本地代码文件中不包含它，所以线上线下就对不上了。）

例如：我这里远程仓库有readme自述文件，自己的本地仓库没有，就会报错，

使用git pull –rebase origin master（分支名），即可将线上、线下代码进行合并

然后再进行push：git push origin master

git pull origin 【branch-name】远程仓库拉取

如果远程仓库和本地仓库代码一直，则不需要拉取
